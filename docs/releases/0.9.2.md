# Videorc 0.9.2 Beta 1

Videorc 0.9.2 fixes **packaged-app camera/microphone capture** — the 0.9.1
"permissions are on but the camera shows nothing" bug — and is the first release
delivered to users through the in-app updater.

## The fix

Every binary in the bundle is signed with hardened runtime from one shared
entitlements plist, and that plist carried no AV device entitlements. Under
hardened runtime, `com.apple.security.device.camera` /
`com.apple.security.device.audio-input` are mandatory for device access — a TCC
grant in System Settings cannot override their absence, so camera/mic capture
(backend → ffmpeg `-f avfoundation`, plus native AVFoundation paths and even
Electron's own `askForMediaAccess`) was flatly denied in packaged builds while
working in dev (not hardened-signed). Screen recording was unaffected — it has
no entitlement requirement (TCC-only).

- `apps/desktop/build-resources/entitlements.mac.plist` now carries both device
  entitlements; the same plist signs the app, `videorc-backend`,
  `native_preview_host_helper`, `ffmpeg`, and `ffprobe` in both signing paths.
- `release:validate:macos` now **fails closed** if the app or any bundled tool
  is missing the device entitlements, so this class of regression cannot ship
  silently again.

If camera still appears blocked after updating on a machine that ran 0.9.1:
System Settings → Privacy & Security → Camera → toggle Videorc off/on once (the
pre-fix build may have cached an auto-denial), or
`tccutil reset Camera dev.theorcdev.videorc`.
