# Videorc 0.9.7 Beta 1

Videorc 0.9.7 is the **capture fidelity + takeover** release: full-Retina
window recordings, the tear-free native preview ring, the takeover rework
(Assets management + Studio on-air switch), offline premium, and a
Studio/Settings declutter pass.

## Scope

- **Preview resolution + tearing** (PT1–PT5) — window captures request real
  pixels instead of points (recordings were half-res on Retina); the
  compositor renders into a 3-slot IOSurface target ring with a per-slot
  helper import cache, eliminating cross-process tearing (present benchmark
  120fps); change-triggered sizing diagnostics on both lanes; dock probe
  asserts drawable == slot points × display scale. Acceptance:
  `docs/acceptance/2026-07-05-preview-res-tearing.md`.
- **Takeover rework** (T1–T3) — the takeover grid moved from the Scene page
  to Assets (management: upload/rename/reorder/delete); the on-air switch is
  a chip block in the Studio session panel (live-safe, one home per
  control); user-facing copy renamed Screens → takeovers (internal
  ids/RPCs/protocol unchanged); takeover activation drives the standby
  preview; an active takeover waives hidden-source checks at recording
  start; images served via `videorc-asset://screen`.
- **Fork protection / entitlements** (FP1–FP4) — `VIDEORC_PREMIUM_FEATURES`
  is downgrade-only (no env unlock in any build, permanent regression test);
  Ed25519-signed entitlement token minted by the web API, verified against
  the compiled-in pubkey and persisted, so premium survives offline for 7
  days; TRADEMARK.md brand policy; docs truth pass.
- **F-021** — initialize CoreGraphics before any window-server call and
  reject window frames SkyLight cannot map before filter init (startup crash
  path at recording start).
- **Studio session panel** — title/Status/Mode rows and subtitles removed
  (status lives on the Preview header badge); Record/Stream promoted to
  full-width glassy hero controls (glass-shine, h-11, brand-red record
  accent); Stream hover brightens via color-mix.
- **Scene page simplification** — the Background overrides section is gone
  and the (never actually per-scene) override layer with it;
  `effectiveSceneBackground` renders asset style defaults exactly; stale
  persisted overrides are dropped at reconcile. Assets → Active background
  is the single editing home.
- **Settings** — launch-at-login removed end-to-end (UI, preload bridge,
  types, IPC handlers); lower region rebalanced into explicit columns
  (Appearance/Support/About stack beside full-height Shortcuts).
