# Videorc 0.9.21 Beta 1

Videorc 0.9.21 ships plan 030: X broadcasts are now verified watchable, the
X lifecycle is fully observable, and sources that repeatedly fail playback
are retired automatically.

## Scope

- **Playback verification** (`x_live.rs`, `main.rs`, plan 030 S2) — the
  create-broadcast `video_access.hls_url` feeds a bounded pre-publish gate
  (45 s, so the announcement post links to working video when X provisions
  in time) and a post-publish watcher (up to 5 min) that reports
  `x-playback-verified` (with time-to-playable), `x-playback-pending`
  (90 s warning), or `x-playback-unavailable`.
- **Lifecycle observability** (S1) — `x-source-prepared`,
  `x-broadcast-published`, `x-broadcast-ended`, `x-*-failed`, and
  `x-source-compatibility` land in session logs / support bundles;
  FFmpeg progress spam demoted to debug tracing (it evicted the 200-entry
  log ring in ~60 s during the 2026-07-08 incident).
- **Source health** (S3) — per-source playback outcomes persisted
  (`app_settings.xSourceHealth`); retirement after 2 consecutive failed
  sessions; prepare ladder env-override → non-retired name match → adopt a
  measured healthy in-region source → create; retired sources deleted
  best-effort when idle.
- **Renderer** (S4) — `streamTargets.x.playback` events drive the X target
  row (share URL, warning copy) with one-shot toasts.
