# Videorc 0.9.10 Beta 1

Videorc 0.9.10 is the **self-updating + output-path safety** release. Two
changes, both urgent.

## Scope

- **Automatic update checks** — `initAutoUpdater` gate flipped from opt-in
  (`VIDEORC_ENABLE_AUTO_UPDATE=1`, dead flag from before the feed existed) to
  opt-out (`VIDEORC_DISABLE_AUTO_UPDATE=1`). Packaged builds check on every
  launch, download in the background, notify when ready, apply on the next
  quit (never a forced restart). The sidebar chip reads the same shared
  status; the renderer pulls it on mount so pre-window check results land.
  This makes every FUTURE release self-propagating; 0.9.9-and-earlier still
  need one manual check.
- **Output directory `~` expansion** (incident 2026-07-06) — a Settings value
  of `~/Movies/…` was passed to the recorder as a literal relative path;
  ffmpeg resolved it against the backend cwd and wrote THREE recordings
  inside the signed app bundle (`/Applications/Videorc.app/Contents/~/…`),
  invisible to the Library and breaking the code signature. Settings
  validation had expanded the tilde (green chip) while the recorder did not —
  the mismatch. The backend now expands a leading `~` (recording + import)
  and refuses still-relative paths with an actionable error. Owner's machine
  remediated by hand: files moved to the real folder, DB rows repaired,
  bundle `~` dir removed, `codesign --verify --deep --strict` OK.
